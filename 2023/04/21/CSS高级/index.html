BFC概念及特性

  1. 概念

    BFC-Block Formatting Context(块级格式化上下文)

  2. 如何触发BFC

    a)float: left或right

    b)overflow: hidden/scroll/auto

    c)display: table-cell,table-caption,inline-block,flex

    d)position: absolute/fixed

  3. BFC作用

  • 解决上下margin重叠问题

    给谁触发：给其中一个元素外层包含一个父级，并触发BFC，

    效果：同时设置上下margin的两个元素，margin值不会发生重叠

  • 解决外边距合并

    给谁触发：父级触发了BFC，

    效果：子元素设置的margin值不会传递给父级

  • 解决高度塌陷问题

    给谁触发：父级触发了BFC，

    效果：浮动的子元素会被父元素计算高度

  • 实现多列布局

    给谁触发：非浮动元素触发了BFC

    效果：非浮动元素不会覆盖浮动元素的位置

  • 与浮动元素相邻的已生成BFC的元素不能与浮动元素互相覆盖。利用该特性可以作为多列布局的一种实现方式

link和import引入的区别

  1. link是html标签，除了可以引入css文件外，还能引入其他文件，@import属于css范畴，只能引入css文件（老祖宗区别）
  2. link引入的css文件和页面同时加载，@import引入的css文件在页面加载完成后载入（加载顺序不同）
  3. link引入css文件无任何兼容性问题，@import是css2.1提出，低版本IE浏览器不支持（浏览器支持不同）
  4. link支持js控制DOM改变样式，@import不支持。（是否支持js）

扩展：get和post区别

a. get主要用来获取数据，post主要用来发送数据

b. get发送数据时，会显示在url地址栏中，post直接通过服务器发送数据，用户看不到这个过程

c. get可以发送的数据量较小，post可以发送的数据量较大

d. get安全性较低，post安全性较高

★单行文本省略号设置，必须满足以下四个条件：

1）设置一定宽度 width:数值

2）设置文本强制在一行显示 white-space: nowrap;

3）文本隐藏不显示 overflow: hidden;

4）显示省略号 text-overflow: ellipsis;

盒模型

  • 一个盒子由四个部分组成：content、padding、border、margin

  • 分类

    • W3C 标准盒子模型：width/height 只是内容高度，不包含 padding 和 border值
    • IE 怪异盒子模型：width/height 包含了 padding和 border值

  • 改变盒子模型

不同元素隐藏的区别

  • display: none; 从文档流消失，不占据文档空间，但是还存在DOM树中

  • visibility: hidden; 和 opacity: 0; 还是会占据文档空间

  • display: none; 和 visibility: hidden; 绑定的事件不会触发

  • opacity: 0; 的元素绑定的事件还是会触发事件

相对定位，绝对定位，固定定位三者的区别

1）参照物不同

相对定位 自身

绝对定位 离他最近的有定位属性的包含块

固定定位 可视窗口

2）是否保留位置

相对定位 不脱离文档流 位置保留

绝对定位 固定定位 会脱离文档流 位置会被其他元素占据

注意：相对浏览器时 absolute是针对整个页面 fixed针对可视窗口

高度塌陷问题（常见的几种清除浮动的方案及其优缺点）

  • 原因：父元素高度自适应，子元素浮动后，导致父元素高度为0，叫做高度塌陷问题

  • 解决方案：

  1. 给父元素设置高度

​ 缺点：设置固定高度以后父元素高度就不能自适应了

  1. 给父元素设置overflow:hidden;

    • 原理：由于父元素高度自适应，子元素浮动，脱离文档流不再占据空间，导致父元素高度为0，当给父元素设置overflow:hidden;后，触发BFC机制，此时闭合了子元素浮动，子元素重新参与父元素高度计算，父元素高度不再为0；

    • 缺点：显示在父元素框之外的部分会被隐藏

​ 缺点：造成代码冗余

常见浏览器兼容性问题

  1. 表单元素行高对齐方式不一致

    解决方案：

    1）input{float:left;}

    2）给input添加一致的vertical-align属性值（middle/top/bottom都行）

  2. 图片默认有间隙

    解决方案：

    1）img{display：block;} 适合给单个img设

    2）img{float:left;} 适合多个图片时候用 左右上下都能去掉 父元素要加overflow:hidden

  3. 在IE6中设置较小高度的容器时，显示不正确（都以高度大于19px显示）

    解决方案：overflow:hidden;

  4. IE6不能识别min-height属性

    解决方法：min-height:100px; _height：100px; （添加下划线的css属性只有IE6识别，其他浏览器不识别）

  5. IE8及以下浏览器不能识别opacity属性

    解决方案：opacity:0.5; filter:alpha(opacity=50)；(两句话都要写)

水平垂直居中问题

  1. 已知大小的元素在屏幕窗口水平垂直都居中
    元素{width:100px;height:100px;position:fixed;left:50%;top:50%;margin-left:-50px;margin-top:-50px;}

  2. 未知大小的元素在屏幕窗口水平垂直居中
    元素{position:fixed;left:0;right:0;top:0;bottom:0; margin:auto;}

  3. 已知大小的子元素在父元素中水平垂直居中
    父元素{position:relative;}
    子元素{width:100px;height:100px;position:absolute;left:50%;top:50%;margin-left:-50px;margin-top:-50px;}

  4. 未知大小的子元素在父元素中水平垂直居中
    父元素{position：relative}
    子元素{position:absolute;left:0;right:0;top:0;bottom:0; margin:auto;}

  5. 使用flex布局的方式实现已知或未知大小的元素在屏幕窗口水平垂直都居中

    html,body{height:100%; 让html，body的高度等于屏幕窗口的高度（高度自适应）}

    body{display:flex;justify-content:center;align-items:center;}

  6. 使用flex布局的方式实现已知或未知大小的子元素在父元素中水平垂直都居中

    父元素{display:flex;justify-content:center;align-items:center;}

flex为1重要应用

  • flex默认值是 flex:0 1 auto;

即：

flex-grow的默认值为0，即使父元素存在剩余空间也不放大

flex-shrink的默认值为1，父元素空间不足时，flex项目等比例缩小

flex-basis默认为auto时，先看有没有width,没有width就根据内容撑开

  • flex:1是 flex:1 1 0% 的简写

flex-grow值为1时，flex项目等比例放大占满flex容器（等比例分配剩余空间）

flex-shrink的默认值为1，父元素空间不足时，flex项目等比例缩小

flex-basis为0时，表现上，该item会折叠到最小的宽度（就是全是剩余空间）

  1. 应用1：分配剩余空间给某个flex项目

  2. 应用2：等比例分配剩余空间

响应式布局

1.概念

响应式是指根据不同设备浏览器分辨率或尺寸来展示不同页面结构、行为、表现的设计方式（考试）

2.原理

响应式设计的基本原理时通过媒体查询检测不同的设备屏幕尺寸做处理，页面头部必须有meta声明viewport

  （快捷键meta:vp）

3.优缺点

优点：解决了多设备屏幕适应问题，灵活性更强

缺点：工作量大，加载时间变长
