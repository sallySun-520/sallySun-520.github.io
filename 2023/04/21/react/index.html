React基础

React基础

  1. 用于动态构建用户界面的JavaScript库（只关注于视图）
  2. 由Facebook开源

  • React的特点

    1. 声明式编码
    2. 组件化编码
    3. JSX语法
    4. 高效（优秀的Diffing算法）
    5. 单向响应的数据流会比双向绑定的更安全，速度更快

  • React高效的原因

    1. 使用虚拟DOM，不总是直接操作页面真实的DOM
    2. DOM Diffing算法，最小化页面重绘

模块与组件、模块化与组件化的理解

  • 模块：复用js

  • 组件：用来实现局部功能效果的代码和资源的集合（html/css/js/image等），可以复用编码、简化、提高运行效率

Diff算法

  • 是什么?

    更高效地通过对比新旧Virtual DOM来找出真正的Dom变化之处

  • 算法的基本思想

  1. 新旧虚拟DOM对比的时候，Diff算法比较只会在同层级进行, 不会跨层级比较。

  2. Diff算法是:深度优先算法。 时间复杂度:O(n)

  • 什么是虚拟DOM？

    虚拟DOM是一个对象，一个用来表示真实DOM的对象

  • 原理

    react中diff算法主要遵循三个层级的策略：：

    1. Dom树层，只针对同层的节点进行比较，只有删除、创建操作，没有移动操作
    2. 组件层，如果是同一个类的组件，则会继续往下diff运算，如果不是一个类的组件，那么直接删除这个组件下的所有子节点，创建新的
    3. 节点层：对于同级的节点，根据key进行增删改移动
    • 在旧的虚拟DOM中找到与新的虚拟DOM相同的key
      • 内容没有发生变化,就直接使用原来的真实DOM
      • 内容发生改变,就替换掉之前旧的虚拟DOM,生成新的真实DOM
    • 在旧的虚拟DOM中未找到与新的虚拟DOM相同的key
      • 直接生成新的真实DOM

  • 参考链接:面试官：说说React diff的原理是什么？ | web前端面试 - 面试官系列 (vue3js.cn)

使用 index 作为 key 可能引发的问题

  • 若对数据进行逆序添加、逆序删除等破坏顺序的操作，会进行没有必要的真实 DOM 更新。界面效果没问题，但效率低下。

  • 如果结构中包含输入类的 DOM（如 input 输入框） ，则会产生错误的 DOM 更新。

  • 若不存在对数据逆序添加、逆序删除等破坏顺序的操作，则没有问题。

受控组件和非受控组件

  • 受控组件是指组件状态由React状态控制，并且只能通过使用 setState() 来更新

  • 非受控组件更像是传统的HTML表单元素，数据存储在DOM中，而不是组件内部，获取数据的方式是通过ref引用

  • 参考链接：https://juejin.cn/post/7074541504869433352

React组件通信

组件传递的方式有很多种，根据传送者和接收者可以分为如下：

  • 父组件向子组件传递

  • 子组件向父组件传递

  • 兄弟组件之间的通信:共同父组件

  • 父组件向后代组件传递:Context

  • 非关系组件传递:redux
