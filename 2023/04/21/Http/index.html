一、http协议

1.http概念

http 是我们前后台交互的时候的传输协议（即超文本传输协议）

  • 特点：
  1. 只能由前端发起
  2. 一次只能说一件事
  3. 只能传递字符串

2.工作流程

  1. 建立连接：TCP三次握手

  2. 发送请求

  • 请求报文

1.请求行

POST /user HTTP/1.1
· POST 请求方式
· /user 请求 URL（不包含域名）
· HTTP/1.1 请求协议版本

2.请求头

user-agent: Mozilla/5.0 表示产生请求的浏览器信息
accept: application/json 表示客户端希望接受的数据类型
Content-Type: application/x-www-form-urlencoded 表示客户端发送的实体数据格式,即请求体的数据格式
Host: 127.0.0.1 表示请求的主机名（IP）
cookie
3.请求空行（请求头和请求主体之间要留一个空行）

4.请求体（本次请求携带的数据）

GET 请求是没有请求体数据的
POST 请求才有请求体数据的

  1. 响应请求
  • 响应报文

1.状态行

HTTP/1.1 200 OK
·HTTP/1.1 服务器使用的 HTTP 协议版本
·200 响应状态码
·OK 对响应状态码的简单解释

  1. 响应头

Date: Jan, 14 Aug 2019 12:42:30 GMT // 服务器时间
Server: Apache/2.4.23 (Win32) OpenSSL/1.0.2jPHP/5.4.45 // 服务器类型
Content-Type: text/html // 服务端给客户端的数据类型,即响应体的数据格式
Content-Length: 11 // 服务端给客户端的数据长度

  1. 响应体

hello world // 服务端给客户端的响应数据

  1. 断开连接：TCP四次挥手

3.常见的HTTP响应状态码

  • 100~199：请求继续
  • 200~299：请求成功
    • 200： 标准请求成功
  • 300~399：重定向
    • 301：永久重定向
    • 302： 临时重定向
  • 400~499：客户端错误
    • 404： 服务器找不到你请求的 URL
  • 500~599：服务器错误
    • 500： 服务器内部错误

4.常见的HTTP请求方式

  1. GET： 一般用于获取一些信息使用（获取列表）
  2. POST： 一般用于发送一些数据给服务端（登录）

二、cookie

  • 概念

HTTP Cookie（也叫 Web Cookie 或浏览器 Cookie）是服务器发送到用户浏览器并保存在本地的一小块数据，它会在浏览器下次向同一服务器再发起请求时被携带并发送到服务器上。通常，它用于告知服务端两个请求是否来自同一浏览器，如保持用户的登录状态。Cookie 使基于无状态的HTTP协议记录稳定的状态信息成为了可能。

  • cookie的特点
  1. 每个域名cookie限制为50条左右
  2. 容量小,单条cookie不超过4KB
  3. 有有效期
  4. 有域名限制,谁设置谁读取，谁存储谁访问
  5. 以字符串形式存储,存储形式key1=value1;key2=value2
  6. 前后端都可以操作
  7. 发送任何一个请求时候会自动携带cookie里面的内容

cookie、localStorage和 SessionStorage的区别

对比项目 cookie localstorage sessionStorage
数据存储时间 可设置失效时间 永久 仅当前会话
容量 <=4kb <=5mb <=5mb
